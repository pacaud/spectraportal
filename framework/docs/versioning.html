Current status

You’re reading docs for v0.1. That means the system is already useful and consistent, but you should expect occasional small refinements.

What “stable” means here

  • Tokens are the contract. Token names should not churn.
  • Classes are conservative. Common classes (like .sp-button) should remain valid.
  • Demos are the truth. If the demos still render correctly, Spectra is functioning as expected.

How to upgrade safely

  1. Pin a versioned CSS URL (don’t use an unversioned “latest” for production).
  2. Open the demo index and spot check key pages (buttons, forms, cards, drawer).
  3. If you override tokens, scan your overrides for renamed variables.
