Core rules

  • Tokens are the source of truth. Spacing, color, type, radius, shadow, motion, z-index.
  • No raw values. If it can’t be expressed with a Spectra token, it doesn’t belong yet.
  • Themes swap values, not structure. Themes override tokens; components don’t change.
  • Framework first, tools second. Tools depend on Spectra; Spectra never depends on tools.

How to use

Load the built CSS from the portal:

<link rel="stylesheet" href="https://spectraportal.dev/framework/cdn/v0.1/spectra.css" />
<link rel="stylesheet" href="/theme.css" />

Docs rule

Docs should not invent styling. If the docs need a style, it probably belongs in Spectra (token/base/util), not as one-off CSS.
